# 线性后台功能目录

## 目标

`admin.html` 先做“会话优先的城主工作台”，不继续堆表单墙。后台下一轮重构前，先按功能用途裁剪：哪些要保留在首屏、哪些放进更多、哪些暂时隐藏。

## 建议保留在首屏

| 功能 | 用途 | 当前状态 | 建议 |
| --- | --- | --- | --- |
| 会话列表 | 看待跟进会话、私信、群聊、世界广场线程 | 可用 | 保留左侧，作为入口主轴 |
| 当前会话 | 阅读当前线程、回复、记录跟进 | 可用 | 保留中间主位 |
| 刷新会话 | 从 gateway 重新拉取当前可见状态 | 可用 | 保留为高频动作 |
| 导出会话 | 导出当前会话或可见历史 | 已接入口 | 保留，但放在当前会话工具区 |
| 去找居民 | 从会话跳转到居民/对象上下文 | 部分可用 | 保留，但标记“上下文动作” |

## 建议保留但收进“当前工具”

| 功能 | 用途 | 当前状态 | 建议 |
| --- | --- | --- | --- |
| 登录与身份 | 城主/居民身份查看、验证码登录 | 可用 | 只在需要时展开 |
| 公告发布 | 发布世界公告或城邦公告 | 已有表单，需 gateway | 收进工具，不抢主位 |
| 安全通告 | 发布安全提示、处理安全状态 | 已有表单，需 gateway | 收进工具，标明影响范围 |
| 举报处理 | 查看/处理世界安全举报 | 已有入口，依赖数据 | 收进工具，保持只读优先 |
| 居民制裁 | 冻结、限制或恢复居民权限 | 高风险动作 | 收进工具，并要求二次确认 |

## 建议暂时隐藏或降级

| 功能 | 原因 | 建议 |
| --- | --- | --- |
| provider 互联接入 | 不是日常会话处理动作 | 放进“高级/互联实验” |
| 城市/房间创建 | 当前主线先稳定单城 IM | 从首屏移除 |
| 世界镜像注册 | 互联扩展功能，容易干扰后台理解 | 从首屏移除 |
| 信任状态深改 | 需要清晰权限和审计，再开放 | 先只读展示 |

## 下一版布局建议

- 左侧：会话列表，只放过滤、搜索、待跟进摘要。
- 中间：当前会话，只放消息、当前对象、输入框、发送状态。
- 右侧：当前工具，只显示与当前会话有关的动作；默认折叠高级治理。
- 顶部：身份、网关、同步状态，保持 3 到 5 个短标签。

## 验收标准

- 用户第一眼能看懂“先处理哪条会话”。
- 每个按钮都标清“可用 / 需网关 / 只读 / 待接入”。
- 首屏不出现大面积治理表单。
- 高风险动作不直接暴露在首屏。
- 后台视觉只服务会话处理，不再抢 IM 主流程。
