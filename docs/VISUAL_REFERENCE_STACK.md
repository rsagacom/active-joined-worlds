# Visual Reference Stack

这份文档是 `lobster-chat` 的固定视觉/交互参考矩阵。

目的只有一个：

- 避免后续界面方向漂移
- 避免把聊天主界面做成总览台/杂乱栏目墙
- 明确 `TUI`、`H5`、`城主共建页`、`世界共建页` 各自该参考什么

## 一、用户已确认的主视觉方向

### 1. 城市公共界面

- 参考图：local visual reference 01
- 语义：
  - 城邦
  - 公共频道
  - 居民区
  - 城主府
  - 传送阵
  - 钟塔
  - 角色资料
- 用途：
  - 默认城市公共聊天室
  - 城邦外页
  - 城主/角色资料页
- 也可作为世界地图页的视觉基线

### 2. 住宅私聊界面

- 参考图：local visual reference 02
- 语义：
  - 住宅场景
  - 左侧极简功能栏
  - 底部消息流
  - 输入框
- 用途：
  - resident 默认住宅私聊页
  - `TUI` 私聊主界面
  - `H5` 私聊页

### 3. 重复图

- 旧本地参考图与房间内界面方向重复，开源包不包含该本机素材。
- 可忽略，不作为独立基线

## 二、参考资源分层

### A. 聊天主界面主参考

这组资源决定“城市公共首页和住宅私聊页分别应该长什么样”。

1. [RPGUI](https://github.com/RonenNess/RPGUI)
   - 用途：`SFC/16-bit RPG` 视觉语言主参考
   - 适合：
     - 房间内界面
     - 木质边框
     - 血条/状态条
     - 复古卷轴、面板、提示框

2. [NES.css](https://github.com/nostalgic-css/NES.css)
   - 用途：8-bit 像素控件语言参考
   - 适合：
     - 按钮
     - 对话框
     - 像素头像框
     - 低复杂度操作件

3. [retro-aim](https://github.com/mattkuda/retro-aim)
   - 用途：复古 IM 交互心智参考
   - 适合：
     - 左侧会话列表
     - 消息收件箱
     - 频道/私聊切换

4. [chat-react-front](https://github.com/Tomilomi/chat-react-front)
   - 用途：像素风聊天界面参考
   - 适合：
     - H5 聊天页
     - 输入框布局
     - 底部消息流

### B. 世界/城邦参考

这组资源决定“城市外 / 世界页 / 大地图”该怎么做。

1. [WorkAdventure](https://github.com/thecodingmachine/workadventure)
   - 用途：多人像素世界/房间同步参考
   - 适合：
     - 世界/城邦空间语义
     - 房间切换
     - 多人在线空间感

2. [Phaser](https://github.com/phaserjs/phaser)
   - 用途：2D Tilemap / 大地图渲染参考
   - 适合：
     - 城邦地图
     - 世界外页
     - 地块与视口逻辑

3. [Mapscii](https://github.com/rastapasta/mapscii)
   - 用途：终端地图表现参考
   - 适合：
     - `TUI` 世界外地图
     - 低带宽大地图
     - 字符图/盲文符号地图

### C. 城主/共建页参考

这组资源决定“城主维护页”和“世界共建页”该怎么分家。

1. [pixelact-ui](https://github.com/pixelact-ui/pixelact-ui)
   - 用途：现代前端技术栈下的像素组件参考
   - 适合：
     - Web 版城主维护页
     - 抽屉化工具区
     - 更现代的共建页实现

2. [98.css](https://github.com/jdan/98.css)
   - 用途：机房/系统室气质参考
   - 适合：
     - 城主维护页
     - 系统监看区
   - 不适合：
     - 默认聊天首页

3. [PSone.css](https://github.com/micah5/PSone.css)
   - 用途：千禧年早期 3D/像素混合视觉参考
   - 适合：
     - 特殊活动页
     - 次级 UI 皮肤
   - 不作为主参考

4. `Discuz` 式信息架构
   - 用途：世界共建页的信息分类方式
   - 原则：
     - 按用途线性分栏
     - 一眼知道每个栏目做什么
     - 不允许杂乱栏目墙

### D. 终端主战场参考

这组资源决定 `TUI` 本体如何做，不是做 Web 入口。

1. [Bubble Tea](https://github.com/charmbracelet/bubbletea)
   - 用途：终端应用交互范式参考

2. [Textual](https://github.com/Textualize/textual)
   - 用途：终端布局与多区组织参考

3. [Blessed-contrib](https://github.com/yaronn/blessed-contrib)
   - 用途：终端多区状态组织参考

4. [Lazygit](https://github.com/jesseduffield/lazygit)
   - 用途：终端产品完成度参考
   - 重点看：
     - 焦点管理

5. [Synchronet](https://github.com/SynchronetBBS/sbbs)

## 三、明确的产品分工

### 默认聊天首页

必须像：

- 城市公共聊天室
- 城里有人
- 可以直接发消息

默认页只保留：

- 顶部轻量切换条
- 中间城市场景
- 右侧角色资料/状态卡
- 底部消息流
- 输入框

不允许：

- 杂乱栏目墙
- 复杂系统分区
- 大量解释文字
- 默认展示世界共建页

### 住宅私聊页

必须像：

- 有人住的私聊住宅
- 左侧极简功能栏
- 中间住宅场景
- 可以直接续聊

不用于：

- 默认城市公共首页

### 城市外页

必须像：

- 城邦外页
- 世界地图
- 居民区/城主府/传送阵这些对象都能一眼认出来

不用于：

- 默认聊天首页

### 城主共建页

必须像：

- 群主维护页
- 少量高频动作
- 一眼明白功能

不允许：

- 复杂城务栏目墙
- 一页塞满城务分区

### 世界共建页

必须像：

- 线性栏目共建页
- 更接近 `Discuz` 式的信息架构

至少要有清晰分栏：

- 世界/城邦
- 城市
- 居民
- 安全
- 镜像/互联
- 日志/任务

## 四、当前执行原则

1. `TUI` 是主战场。
2. `H5` 应尽量显示成 `TUI` 的同构界面，不另起一套独立页面语义。
3. 默认首页始终参考“城市公共界面”。
4. 住宅私聊页单独成页，参考“住宅私聊界面”；世界地图和城邦外页参考“城市公共界面”的外层空间表达。
5. 城主维护页极简。
6. 世界共建页走 `Discuz` 式线性栏目。

## 五、禁止跑偏

以下方向视为跑偏：

- 第一眼像机房工具页
- 第一眼像开发调试工具
- 首页信息量大于消息流和输入区
- 通过长说明文字解释“居民/城主/城市/世界”是什么
- 城主页长得像办公流程首页
- 世界共建页仍然是无分类栏目墙
